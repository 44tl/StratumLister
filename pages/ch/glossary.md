# Glossary

A comprehensive list of technical terms used throughout the Stratum documentation to ensure clarity and precision.

## Attack Surface
The set of all points where an unauthorized user (the "attacker") can try to enter data to or extract data from an environment. Reducing the attack surface is a primary goal of "hardening."

## Backup Image
A byte-for-byte copy of a storage drive or partition. Unlike a file backup, a disk image includes the operating system, boot sector, and all configurations, allowing for a total system recovery in case of catastrophic failure.

## Bloatware
Unnecessary software bundled with an operating system or hardware. This includes trialware, manufacturer-installed utilities, and telemetry-heavy background services that consume system resources without providing meaningful value.

## Debloat
The process of selectively removing or disabling bloatware. **Safe debloating** is surgical, well-documented, and fully reversible, rather than using "all-in-one" scripts that may break system dependencies.

## DDU (Display Driver Uninstaller)
A specialized utility designed to completely remove GPU and audio drivers and their leftovers (registry keys, folders, files, driver store). It is used to ensure a clean environment when troubleshooting driver crashes or switching between hardware vendors (AMD/Nvidia/Intel).

## DNS (Domain Name System)
The decentralized system that translates human-readable hostnames (e.g., `google.com`) into machine-readable IP addresses. Modern DNS settings (like DoH or DoT) can improve privacy by encrypting these requests.

## Hardening
The practice of securing a system by reducing its vulnerabilities. This involves disabling unnecessary services, closing unused ports, applying the principle of least privilege, and configuring defensive settings.

## PBR (Physics Based Rendering)
A method of shading and rendering that provides a more accurate representation of how light interacts with surfaces. Mentioned in technical sections regarding graphics and multimedia performance.

## Restore Point
A snapshot of the Windows Registry and critical system files. While useful for "undoing" recent configuration errors or driver installations, it is not a substitute for a full data backup.

## SHA-256 Checksum
A unique 64-character string generated by a cryptographic hash function. Comparing the SHA-256 checksum of a downloaded file against the official source ensures the file has not been tampered with or corrupted during transit.

## Telemetry
The automated process of collecting and transmitting data from a remote source (your PC) to a service provider (Microsoft, Google, etc.). Most optimization guides focus on disabling non-essential telemetry to improve privacy and reduce background resource usage.

## Threat Model
A structured approach to identifying and prioritizing potential security risks. It asks: "What do I want to protect?", "Who am I protecting it from?", and "How much inconvenience am I willing to tolerate?"
